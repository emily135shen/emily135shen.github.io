PROBLEM SOLVING

Getting Blocked on a simple question

I got blocked on a simple problem in my code the other day trying to fix an error of an image not appearing on my website. The Problem-solving method I used was Pseudocode. This is a method of simplifying programming language used in program design. So instead of writing the code out we write it in words and sentences with no code to help us understand what’s going on. The process of this was really helpful because it simplified the problem and helped me see what wrote wrong in my code. In the process of t his I learn that my image sizing wasn’t right therefore the image wasn’t showing up and I also learnt that writing out your problem or error in a different perspective can be really helpful.

The problem solving techniques and process

  • Pseudocode
  • Trying something
  • Rubber ducky method
  • Reading error messages
  • Console.logging
  • Googling
  • Asking your peers for help
  • Improving your process with reflection

Reflection on techniques and processes

I am familiar with the techniques and process of most of these techniques. These techniques include (googling, trying something new, reading error messages, googling, asking peers for help, improving my process through reflections) and these are the techniques I am confident with. However, I have not used the solving techniques and process of pseudocode and rubber ducky. These are the solving techniques I am not as familiar with however I am starting to use these techniques on my daily coding activities.
